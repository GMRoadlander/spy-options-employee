<task type="auto">
  <name>Task 1: Z-score anomaly detectors</name>
  <files>src/ml/anomaly.py, tests/test_anomaly.py</files>
  <action>
Create `src/ml/anomaly.py` with pure statistical anomaly detection functions:

1. `detect_volume_anomaly(current_volume: int, historical_volumes: list[int], threshold_sigma: float = 2.0) -> dict` — z-score of current volume vs rolling history. Returns `{"is_anomaly": bool, "z_score": float, "threshold": float, "volume": int, "mean": float, "std": float}`. Handle edge cases: empty history → not anomaly, zero std → not anomaly.

2. `detect_iv_anomaly(current_iv: float, historical_ivs: list[float], underlying_change_pct: float, threshold_sigma: float = 2.0) -> dict` — flag when IV moves significantly without a corresponding underlying move. If |IV z-score| > threshold AND |underlying_change_pct| < 1%, it's an anomaly. Returns `{"is_anomaly": bool, "iv_z_score": float, "underlying_change": float, "type": "expansion"|"contraction"|"normal"}`.

3. `detect_volume_oi_anomaly(volume: int, open_interest: int, historical_ratios: list[float], threshold_sigma: float = 2.0) -> dict` — flag when V/OI ratio is unusually high (new positioning). Returns `{"is_anomaly": bool, "ratio": float, "z_score": float, "interpretation": str}`. Interpretation: ratio > 3.0 = "aggressive new positioning", ratio > 1.0 = "active trading", ratio < 0.3 = "quiet".

4. `detect_strike_clustering(chain_volumes: dict[float, int], spot_price: float, top_n: int = 5) -> list[dict]` — identify strikes attracting disproportionate volume. Return top N strikes by volume concentration (volume at strike / total volume). Each entry: `{"strike": float, "volume": int, "concentration": float, "distance_from_spot_pct": float, "type": "call"|"put"|"both"}`.

5. `AnomalyReport` dataclass — aggregates all anomaly results for a given snapshot: `volume_anomalies: list[dict], iv_anomalies: list[dict], voi_anomalies: list[dict], strike_clusters: list[dict], overall_score: float (0-1), timestamp: str`.

6. `scan_chain_anomalies(chain: OptionsChain, historical_data: dict | None = None) -> AnomalyReport` — convenience function that runs all detectors on an OptionsChain. If historical_data is None, returns report with all is_anomaly=False (no baseline to compare against). historical_data format: `{"volume_history": {strike: [volumes]}, "iv_history": {strike: [ivs]}, "voi_history": [ratios]}`.

Tests in `tests/test_anomaly.py`:
- Volume anomaly: normal volume → not anomaly, 3σ spike → anomaly
- IV anomaly: IV spike with no underlying move → anomaly, IV spike with underlying move → not anomaly
- V/OI anomaly: high ratio → anomaly with correct interpretation
- Strike clustering: known volume distribution → correct top strikes
- Edge cases: empty history, zero std, single data point
- scan_chain_anomalies with synthetic OptionsChain

Add ML-based anomaly detection and management to `src/ml/anomaly.py`:

1. `FlowAnomalyDetector` class using scikit-learn's IsolationForest:
   - `__init__(self, contamination: float = 0.05, n_estimators: int = 100, random_state: int = 42)` — 5% expected anomaly rate.
   - `fit(self, feature_matrix: np.ndarray) -> None` — fit on historical flow data. Feature columns: [volume, oi, v_oi_ratio, iv, iv_change, premium, delta, gamma, moneyness]. The caller provides the feature matrix.
   - `predict(self, features: np.ndarray) -> list[dict]` — returns anomaly scores for each row: `{"is_anomaly": bool, "anomaly_score": float (-1 to 1, lower = more anomalous)}`. Uses decision_function() for scores and predict() for labels.
   - `save(path)` / `load(path)` — pickle the fitted model.

2. `AnomalyManager` class:
   - `__init__(self, feature_store: FeatureStore, model_dir: str)`
   - `async update(self, chain: OptionsChain, historical_data: dict | None, ticker: str = "SPX") -> AnomalyReport` — runs z-score detectors + isolation forest (if fitted), computes overall_score (weighted average: z-score anomalies contribute 0.6, isolation forest contributes 0.4), persists anomaly_score to feature store. Returns full AnomalyReport.
   - `async train_forest(self, historical_chains: list[dict]) -> dict` — fit isolation forest on historical flow data. Called periodically (monthly) to update the model baseline. Returns training stats.
   - `async get_current_anomalies(self, ticker: str = "SPX") -> AnomalyReport | None` — returns cached latest report.

The overall_score in AnomalyReport ranges from 0 (normal) to 1 (highly anomalous). Computation:

- Normalize to 0-1 range
- Score > 0.7 = "alert", 0.4-0.7 = "elevated", < 0.4 = "normal"

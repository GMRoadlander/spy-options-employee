  <name>Task 1: Unusual Whales REST client for flow and dark pool data</name>
  <files>src/data/unusual_whales_client.py, tests/test_unusual_whales_client.py, src/config.py</files>
  <action>
Create `src/data/unusual_whales_client.py` following established async client patterns:

1. `UnusualWhalesClient` class:
   - `__init__(self, api_key: str)` — store API key, create rate limiter. Token passed via `Authorization: Bearer {key}` header.
   - `async _request(self, endpoint: str, params: dict | None = None) -> dict` — base request. Base URL: `https://api.unusualwhales.com/api`. Handle 401, 429, timeouts. Reuse aiohttp session.
   - `async get_flow(self, ticker: str = "SPX", limit: int = 100) -> list[dict]` — `GET /stock/{ticker}/option-contracts`. Returns recent options flow: `{"ticker": str, "strike": float, "expiry": str, "type": "call"|"put", "side": "bid"|"ask"|"mid", "premium": float, "volume": int, "open_interest": int, "iv": float, "classification": "sweep"|"block"|"standard"|"golden_sweep", "sentiment": "bullish"|"bearish"|"neutral", "timestamp": str}`.
   - `async get_dark_pool(self, ticker: str = "SPX", limit: int = 50) -> list[dict]` — `GET /stock/{ticker}/dark-pool`. Returns dark pool activity: `{"ticker": str, "price": float, "size": int, "notional": float, "exchange": str, "timestamp": str}`.
   - `async get_flow_summary(self, ticker: str = "SPX") -> dict` — `GET /stock/{ticker}/option-contracts/summary` or aggregate from get_flow. Returns: `{"total_premium": float, "call_premium": float, "put_premium": float, "sweep_count": int, "block_count": int, "golden_sweep_count": int, "net_sentiment": float (-1 to 1), "dark_pool_volume": int}`.
   - `async close(self) -> None` — close aiohttp session.

Add to `src/config.py`:
- `unusual_whales_api_key: str = os.getenv("UNUSUAL_WHALES_API_KEY", "")`

Note: API endpoints and response formats are based on Unusual Whales public API documentation. The actual response structure may differ slightly — the client should handle missing fields gracefully with defaults. Like all other data clients, empty API key → empty results with log warning.

Tests in `tests/test_unusual_whales_client.py`:
- Test _request with mocked aiohttp (success, 401, 429)
- Test get_flow parses response correctly
- Test get_dark_pool parses response correctly
- Test get_flow_summary computation
- Test empty API key → graceful empty results
- Test missing/extra fields in API response handled

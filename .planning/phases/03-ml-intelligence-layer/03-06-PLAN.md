  <action>
Create `src/data/polygon_client.py` following the ORATS client pattern (async aiohttp, rate limiting, error handling):

1. `PolygonClient` class:
   - `__init__(self, api_key: str)` — store API key, create rate limiter (5 req/min for free tier, configurable).
   - `async _request(self, endpoint: str, params: dict | None = None) -> dict` — base request method. Base URL: `https://api.polygon.io`. Add `apiKey` to params. Handle 401 (invalid key), 429 (rate limit), timeouts. Return parsed JSON. Reuse aiohttp session (lazy-init, close on cleanup).
   - `async get_options_chain(self, underlying: str, expiration_date: str | None = None, limit: int = 250) -> list[dict]` — `GET /v3/snapshot/options/{underlying}`. Returns list of option snapshots with greeks, IV, volume, OI. Handles pagination via `next_url`.
   - `async get_option_trades(self, options_ticker: str, date: str, limit: int = 1000) -> list[dict]` — `GET /v3/trades/{options_ticker}?timestamp={date}`. Returns individual option trades (for sweep/block detection). Each trade: price, size, exchange, conditions, timestamp.
   - `async get_option_aggregates(self, options_ticker: str, timespan: str = "day", from_date: str = "", to_date: str = "", limit: int = 120) -> list[dict]` — `GET /v2/aggs/ticker/{options_ticker}/range/1/{timespan}/{from_date}/{to_date}`. Returns OHLCV bars.
   - `async get_news(self, ticker: str = "SPX", limit: int = 50) -> list[dict]` — `GET /v2/reference/news?ticker={ticker}&limit={limit}`. Returns news articles with title, published, source, url. This is used by the sentiment pipeline (Plan 04).
   - `async close(self) -> None` — close aiohttp session.

Add to `src/config.py`:
- `polygon_rate_limit: int = int(os.getenv("POLYGON_RATE_LIMIT", "5"))` — requests per minute

Note: The client should work gracefully when API key is empty string — methods return empty lists with a warning log. This allows the bot to start without Polygon configured.

<task type="auto">
  <name>Task 2: Polygon.io WebSocket client for real-time OPRA</name>
  <files>src/data/polygon_client.py, tests/test_polygon_client.py</files>
  <action>
Add WebSocket streaming to `src/data/polygon_client.py`:

1. `PolygonOptionsStream` class:
   - `__init__(self, api_key: str, tickers: list[str] = ["SPX", "SPY"])` — configures WebSocket connection.
   - `async connect(self) -> None` — connect to `wss://socket.polygon.io/options`, authenticate with API key, subscribe to trade and quote channels for configured tickers. Handle connection errors with exponential backoff (1s, 2s, 4s, max 30s).
   - `async listen(self, callback: Callable[[dict], Awaitable[None]]) -> None` — listen loop that receives messages and calls the callback with parsed trade/quote data. Message types:
     - `T` (trade): `{"type": "trade", "ticker": str, "price": float, "size": int, "exchange": int, "conditions": list, "timestamp": int}`
     - `Q` (quote): `{"type": "quote", "ticker": str, "bid": float, "ask": float, "bid_size": int, "ask_size": int, "timestamp": int}`
   - `async disconnect(self) -> None` — clean disconnect.
   - `_classify_trade(self, trade: dict) -> str` — classify trade as "sweep" (multiple exchanges in rapid succession — tracked via a sliding window of recent trades), "block" (single trade > configurable threshold, default 100 contracts), or "standard". This is a simplified version of commercial flow classifiers.

2. `PolygonFlowAggregator` class:
   - `__init__(self, window_seconds: int = 60)` — aggregates streaming trades into flow summaries.
   - `process_trade(self, trade: dict) -> None` — accumulates trade in current window.
   - `get_flow_summary(self) -> dict` — returns `{"total_volume": int, "total_premium": float, "call_volume": int, "put_volume": int, "sweep_count": int, "block_count": int, "net_premium": float, "largest_trade": dict, "window_start": str}`.
   - `reset(self) -> None` — clear current window.

The WebSocket client is designed for the $199/mo Options plan. If running on the starter plan (delayed data), the stream will still work but with 15-min delay. The client should log the plan level on connect.

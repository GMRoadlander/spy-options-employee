  <name>Task 2: Continuous learning with Bayesian confidence calibration</name>
  <files>src/ml/learning.py, tests/test_learning.py</files>
  <action>
Create `src/ml/learning.py` with signal outcome tracking and confidence calibration:

1. `SignalTracker` class:
   - `__init__(self, store: Store)` — uses existing Store for persistence.
   - `async record_signal(self, signal_type: str, direction: str, strength: float, context: dict) -> str` — creates a signal_log entry (uses existing table from Phase 2-1). Returns signal_id for later outcome recording.
   - `async record_outcome(self, signal_id: str, outcome: str, pnl: float) -> None` — updates signal_log entry with actual outcome. Outcome: "correct", "incorrect", "expired" (no trade taken).
   - `async get_accuracy(self, signal_type: str | None = None, days: int = 30) -> dict` — compute signal accuracy stats: `{"total": int, "correct": int, "incorrect": int, "accuracy": float, "avg_pnl": float, "by_type": dict}`.

2. `BayesianCalibrator` class:
   - `__init__(self, prior_accuracy: float = 0.5, prior_strength: int = 10)` — Beta distribution prior. prior_accuracy=0.5 (uninformative), prior_strength=10 (weak prior = 5 successes, 5 failures equivalent).
   - `update(self, outcomes: list[bool]) -> float` — update posterior with observed outcomes (True=correct, False=incorrect). Returns posterior mean (calibrated confidence). Uses Beta-Binomial conjugate update: posterior_alpha = prior_alpha + successes, posterior_beta = prior_beta + failures.
   - `get_confidence(self) -> float` — returns current posterior mean: alpha / (alpha + beta).
   - `get_credible_interval(self, level: float = 0.95) -> tuple[float, float]` — returns credible interval using scipy.stats.beta.ppf.
   - `save(self) -> dict` / `load(self, state: dict) -> None` — serialize/deserialize state (alpha, beta).

3. `LearningManager` class:
   - `__init__(self, tracker: SignalTracker, store: Store)`
   - `async update_calibration(self, signal_type: str = "all") -> dict` — pull recent outcomes from tracker, update calibrator, return calibrated confidence + credible interval.
   - `async get_model_health(self) -> dict` — aggregate health metrics across all signal types: accuracy by type, calibrated confidence, trend (improving/degrading/stable based on rolling 7-day vs 30-day accuracy).
   - Maintains per-signal-type calibrators, persisted to a `model_calibration` table (add to Store.init()).

Add `model_calibration` table to `src/db/store.py`:
```sql
CREATE TABLE IF NOT EXISTS model_calibration (
    signal_type TEXT PRIMARY KEY,
    alpha REAL NOT NULL DEFAULT 5.0,
    beta REAL NOT NULL DEFAULT 5.0,
    last_updated TEXT NOT NULL
)
```

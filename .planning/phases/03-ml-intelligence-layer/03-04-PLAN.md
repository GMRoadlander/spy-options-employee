<task type="auto">
  <name>Task 1: FinBERT model wrapper and text scoring</name>
  <files>src/ml/sentiment.py, tests/test_sentiment.py, requirements.txt</files>
  <action>
Add `transformers>=4.40` to requirements.txt. Install it.

Note: transformers requires torch (already added in Plan 03). FinBERT uses BERT architecture which runs on CPU efficiently.

Create `src/ml/sentiment.py` with:

1. `SentimentScorer` class:
   - `__init__(self, model_name: str = "ProsusAI/finbert")` — lazy model loading (don't download/load until first use). Store model_name.
   - `_ensure_loaded(self) -> None` — downloads and loads the model + tokenizer on first call. Uses `transformers.pipeline("sentiment-analysis", model=model_name)` for simplicity. Caches the pipeline instance.
   - `score_text(self, text: str) -> dict` — returns `{"label": "positive"|"negative"|"neutral", "score": float (0-1 confidence), "sentiment_value": float (-1 to 1)}`. Map: positive → +score, negative → -score, neutral → 0. Handle empty string → neutral 0.0.
   - `score_batch(self, texts: list[str]) -> list[dict]` — batch scoring for efficiency. Uses pipeline's batch mode. Filter out empty strings.
   - `compute_aggregate(self, scores: list[dict]) -> dict` — aggregate multiple scores into: `{"mean_sentiment": float, "positive_pct": float, "negative_pct": float, "neutral_pct": float, "sentiment_velocity": float | None}`. Velocity requires at least 2 scores with timestamps (not computed here, handled by manager).

The model downloads ~250MB on first use. Cache in the default Hugging Face cache dir (`~/.cache/huggingface/`). For Docker deployment, this should be pre-downloaded in the build step or mounted as a volume.

Tests in `tests/test_sentiment.py`:
- Score positive financial text → positive label
- Score negative text → negative label
- Score neutral text → neutral label
- Batch scoring returns correct count
- Empty string handling
- Aggregate computation with known inputs
- sentiment_value sign matches label

Important: Tests should mock the transformers pipeline to avoid downloading the 250MB model in CI. Use `unittest.mock.patch` to mock the pipeline call and return synthetic scores.

Create `src/data/news_client.py` with a simple async news fetcher:

1. `NewsClient` class:
   - `__init__(self, polygon_api_key: str = "")` — supports Polygon.io news API if key provided. Falls back to a basic free news source.
   - `async fetch_headlines(self, ticker: str = "SPX", limit: int = 50) -> list[dict]` — fetches recent news headlines. Returns list of `{"title": str, "published": str, "source": str, "url": str}`.
   - If Polygon API key is available: `GET https://api.polygon.io/v2/reference/news?ticker={ticker}&limit={limit}&apiKey={key}`
   - If no API key: return empty list with a log warning (sentiment will be unavailable until a news source is configured). Do NOT hardcode any alternative news scraping — that's brittle and may violate ToS.
   - Rate limiting: max 5 requests/minute for Polygon free tier.

Add to `src/config.py`:
- `polygon_api_key: str = os.getenv("POLYGON_API_KEY", "")` — same key will be used for full Polygon integration in Plan 06
- `news_lookback_hours: int = int(os.getenv("NEWS_LOOKBACK_HOURS", "24"))` — how far back to fetch news

Add `SentimentManager` class to `src/ml/sentiment.py`:
1. `__init__(self, scorer: SentimentScorer, news_client: NewsClient, feature_store: FeatureStore)`
2. `async update(self, ticker: str = "SPX") -> dict | None` — fetch recent headlines, score them, compute aggregate, compute velocity (compare to yesterday's aggregate from feature store), persist sentiment_score to feature store. Returns aggregate dict or None if no headlines found.
3. `async get_current_sentiment(self, ticker: str = "SPX") -> dict | None` — read latest sentiment from feature store.
4. `async get_sentiment_history(self, ticker: str = "SPX", days: int = 30) -> list[dict]` — read sentiment time series from feature store.

The rolling sentiment index is simply the daily aggregate sentiment_value persisted to the feature store. Velocity is computed as today's sentiment minus yesterday's sentiment.

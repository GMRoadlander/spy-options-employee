  <action>
Add `torch>=2.0` to requirements.txt. Install it (CPU-only version to save space: `pip install torch --index-url https://download.pytorch.org/whl/cpu`).

Create `src/ml/volatility.py` with:

**Data preprocessing:**
1. `VolDataset(torch.utils.data.Dataset)` — takes a DataFrame with columns [realized_vol, vix, vix_slope, regime_state, hurst] and target columns [rv_1d, rv_5d]. Uses a sliding window of `lookback` days (default 60) to create input sequences. Normalizes features using provided mean/std (StandardScaler-style, but manual for portability).

2. `prepare_training_data(returns: pd.Series, vix: pd.Series, vix_slope: pd.Series | None, regime: pd.Series | None, hurst: pd.Series | None, lookback: int = 60) -> tuple[VolDataset, dict]` — computes realized vol (20-day rolling std × √252) as primary feature, constructs forward-looking targets (1-day and 5-day realized vol), drops NaN rows, computes normalization stats, returns dataset + stats dict.

**Model:**
3. `VolLSTM(nn.Module)` — simple LSTM:
   - Input: `n_features` (default 5: rv, vix, vix_slope, regime, hurst)
   - LSTM: 2 layers, hidden_size=64, dropout=0.2
   - FC head: hidden → 32 → 2 (1d forecast, 5d forecast)
   - Forward pass: take last hidden state, pass through FC
   - Keep it simple — no attention, no bells and whistles. LSTM baseline first.

4. `VolForecaster` class:
   - `__init__(self, lookback: int = 60, hidden_size: int = 64, n_features: int = 5)`
   - Holds model, normalization stats, device config
   - CPU-only inference (set device = 'cpu')
   - `predict(self, recent_features: np.ndarray) -> dict` — returns `{"vol_1d": float, "vol_5d": float}`. Input is (lookback, n_features) array. Normalizes using stored stats, runs forward pass, denormalizes output.
   - `save(path)` / `load(path)` — save model state_dict + normalization stats + hyperparams as a single dict via torch.save

Tests in `tests/test_volatility.py`:
- VolDataset creates correct sequence shapes
- VolLSTM forward pass produces correct output shape
- prepare_training_data handles missing features gracefully (None → zeros)
- VolForecaster predict returns correct dict structure
- Save/load roundtrip preserves predictions
- Edge case: insufficient lookback data raises ValueError

<task type="auto">
  <name>Task 2: Training pipeline and feature store integration</name>
  <files>src/ml/volatility.py, src/ml/feature_store.py, tests/test_volatility.py</files>
  <action>
Add training and management capabilities to `src/ml/volatility.py`:

1. `train(self, dataset: VolDataset, epochs: int = 100, lr: float = 0.001, patience: int = 10, val_split: float = 0.2) -> dict` method on VolForecaster:
   - Split dataset temporally (last val_split% for validation — NOT random split, preserves time ordering)
   - Train with Adam optimizer, MSE loss
   - Early stopping based on validation loss (patience epochs without improvement)
   - Return training stats: `{"train_loss": float, "val_loss": float, "epochs_trained": int, "best_epoch": int}`
   - Print progress every 10 epochs
   - This runs on CPU — training on 5 years of daily data (~1260 rows) with lookback=60 produces ~1200 samples, which trains in seconds on CPU

2. `VolManager` class (mirrors RegimeManager pattern):
   - `__init__(self, feature_store: FeatureStore, model_dir: str)`
   - `async initialize(self, returns: pd.Series, vix: pd.Series, ...) -> dict` — prepare data, train model, save checkpoint, return training stats
   - `async update(self, latest_features: dict, ticker: str = "SPX") -> dict` — run inference with recent data, persist vol_forecast_1d and vol_forecast_5d to feature store, return predictions
   - `async get_current_forecast(self, ticker: str = "SPX") -> dict | None` — read latest forecast from feature store
   - Keeps rolling window of recent features in memory (last `lookback` observations)
   - On first load, tries to load saved model from disk; if not found, returns None (needs training)

3. `evaluate(self, test_dataset: VolDataset) -> dict` method on VolForecaster:
   - Run predictions on test set
   - Return metrics: MAE, RMSE, directional accuracy (% of times forecast correctly predicted vol increase/decrease)
   - This is for walk-forward validation of the model

The training pipeline is designed to be run manually (or via a training script) rather than automatically on the droplet. The droplet only runs inference. Training requires:
- Historical SPX returns + VIX (from ORATS)
- Optional: regime state (from HMM), Hurst (from features.py)

- VolManager update persists to feature store

- Evaluate returns correct metric structure

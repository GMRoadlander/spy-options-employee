<task type="auto">
  <name>Task 2: Core feature computation functions</name>
  <files>src/ml/features.py, tests/test_features.py</files>
  <action>
Create `src/ml/features.py` with pure computation functions (no I/O, no async — just math):

1. `compute_iv_rank(current_iv: float, iv_history: list[float]) -> float` — percentile rank of current ATM IV vs trailing 252-day history. Returns 0-100. Handle edge cases: empty history returns 50.0, single value returns 50.0.

2. `compute_iv_percentile(current_iv: float, iv_history: list[float]) -> float` — percentage of days IV was BELOW current level. Distinct from IV rank (which uses high/low range). Returns 0-100.

3. `compute_skew_25d(chain: OptionsChain) -> float` — 25-delta risk reversal: IV(25Δ put) - IV(25Δ call) for nearest expiry. Use linear interpolation between surrounding deltas to find 25Δ strikes. Return 0.0 if insufficient data. A positive skew means puts are more expensive (normal for equities).

4. `compute_term_structure_slope(chain: OptionsChain) -> float` — ATM IV of 2nd nearest expiry minus ATM IV of nearest expiry, normalized by DTE difference. Positive = contango (normal). Use nearest-to-ATM strike IV. Return 0.0 if fewer than 2 expirations.

5. `compute_rv_iv_spread(returns: list[float], current_atm_iv: float, window: int = 20) -> float` — Realized vol (annualized std of trailing `window` daily returns × √252) minus implied vol. Negative = vol overpriced (edge for sellers = variance risk premium). Return 0.0 if insufficient data.

6. `compute_hurst_exponent(prices: list[float], max_lag: int = 20) -> float` — R/S (rescaled range) analysis. For each lag τ from 2 to max_lag: compute mean rescaled range. Fit log(R/S) vs log(τ) via OLS. Hurst = slope. H < 0.5 = mean-reverting, H > 0.5 = trending, H ≈ 0.5 = random walk. Return 0.5 if insufficient data (< max_lag + 1 prices).

Each function should be stateless, accept numpy arrays or Python lists, and return a single float. Use numpy for vectorized math. Do NOT use external packages beyond numpy/scipy (no `hurst` package — R/S analysis is ~20 lines).

Create comprehensive tests in `tests/test_features.py`:
- IV rank: known history [10, 20, 30, 40, 50] with current 35 → rank = 60
- IV percentile: same setup, verify vs manual calculation
- Skew: synthetic chain with known 25Δ put/call IVs
- Term structure: synthetic chain with 2 expirations, verify slope sign
- RV/IV spread: known returns series, verify annualization
- Hurst: pure trend (H → 1.0), pure mean-reversion (H → 0.0), random walk (H → 0.5)
- Edge cases: empty inputs, single values, insufficient data

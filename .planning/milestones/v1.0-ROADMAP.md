**Status:** SHIPPED 2026-02-24

## Overview

From options alert bot to permanent SPX research + paper trading platform. Four phases built the complete pipeline: real-time data feeds, strategy research engine with rigorous anti-overfitting, ML intelligence layer, and paper trading validation with realistic simulation.

## Phases

**Details:**

- Strategy lifecycle: IDEA → DEFINED → BACKTEST → PAPER → RETIRED
- Anti-overfitting: WFA (12mo IS / 3mo OOS), CPCV (PBO < 0.50), DSR (p < 0.05), Monte Carlo (5th-percentile Sharpe > 0)

## Milestone Summary

**Decimal Phases:**

- Phase 2.1: Code Review Fixes (inserted after Phase 2 for security + quality fixes)

**Key Decisions:**

- Evolve existing bot, not rebuild (validated — preserved 186 tests)
- Tastytrade as real-time data source (free with account, dxfeed WebSocket)
- ORATS for historical backtesting ($99/mo, 2007+ history)
- YAML strategy templates + NL parsing via Claude (not custom DSL)
- Anti-overfitting pipeline mandatory (WFA + CPCV + DSR + Monte Carlo)
- HMM for regime detection (highest-ROI ML model)
- LLMs in research layer only, not execution path (2-30s latency)
- Paper trade to validate (30+ trades required)
- Permanent paper trading platform (no autonomous trading — decided 2026-02-23)
- Structured Claude reasoning over multi-agent SDK (80% of value, simpler)
- Optopsy + custom Python for backtesting (purpose-built for options)

**Issues Resolved:**

- SSRF on Polygon.io next_url pagination
- Information leakage in Discord error messages
- Feature store UPSERT column clobbering
- Pickle/torch model loading security (HMAC + weights_only)
- ML scheduler collision (staggered to 16:10 ET)
- Dark pool ratio denominator zero-division
- Hurst exponent small-sample stability
- 23x deprecated datetime.utcnow() replacements

**Issues Deferred:**

- None — all discovered issues were fixed at point of discovery

**Technical Debt Incurred:**

- Model artifacts on disk require manual backup management (.gitignore blocks ml_features_dir)
- Hurst exponent small-sample upward bias (calibrated but edge cases possible)
- OPRA WebSocket reconnect backoff may need tuning for production Polygon rate limits
- Sentiment velocity uses single-day window (could expand to 3-5 day rolling)
